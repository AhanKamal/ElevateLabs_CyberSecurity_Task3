## Major Vulnerabilities Identified

### 1. SSL Medium Strength Cipher Suites Supported (SWEET32)

**Severity:** High

#### Description

The scan detected support for medium-strength SSL/TLS cipher suites using the 3DES encryption algorithm. These cipher suites are vulnerable to the SWEET32 attack, which can weaken encrypted communications.

#### Risk

Attackers may potentially exploit weak encryption algorithms to compromise the confidentiality of encrypted network traffic.

#### Recommended Mitigation

- Disable 3DES cipher suites.
- Use stronger cipher suites such as AES-GCM.
- Remove outdated SSL/TLS configurations.
- Regularly update cryptographic settings.

---

### 2. SSL Certificate Cannot Be Trusted

**Severity:** Medium

#### Description

The SSL certificate presented by the service could not be verified as trusted.

#### Risk

Users may be vulnerable to man-in-the-middle (MITM) attacks if certificate trust cannot be established.

#### Recommended Mitigation

- Use certificates issued by trusted Certificate Authorities (CA).
- Ensure proper certificate chain configuration.

---

### 3. SSL Self-Signed Certificate

**Severity:** Medium

#### Description

The service is using a self-signed SSL certificate instead of a certificate issued by a trusted CA.

#### Risk

Self-signed certificates do not provide third-party trust validation and may trigger security warnings.

#### Recommended Mitigation

- Replace self-signed certificates with CA-issued certificates.
- Implement proper certificate management practices.

## Conclusion

A vulnerability assessment was successfully performed on the localhost system using Nessus Essentials. The scan completed successfully and identified 25 findings, including one high-severity vulnerability and two medium-severity vulnerabilities related to SSL/TLS security configurations.

This task provided practical experience in vulnerability assessment, vulnerability analysis, severity classification, and remediation planning using an industry-standard security scanning tool.
